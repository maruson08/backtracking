# 4×4 Sudoku Algorithm Visualization

4×4 Sudoku를 기반으로 여러 탐색 알고리즘의 차이를 시각적으로 비교하는 교육용 웹 애플리케이션입니다.

이 프로젝트의 목적은 단순히 스도쿠를 해결하는 것이 아니라,

> "탐색 전략의 차이가 실제 탐색량에 어떤 영향을 주는가"

를 확인하는 것입니다.

# Algorithms

## 1. Naive Search

### 개념

가장 기본적인 완전 탐색 방식입니다.

특별한 최적화 없이 빈 칸을 순서대로 선택하고,
숫자 1~4를 차례대로 시도합니다.

동작 과정:

```

빈 칸 선택
↓
1~4 숫자 시도
↓
가능하면 진행
↓
다음 칸 탐색

```

---

### 특징

- 후보 목록 계산 없음
- 변수 선택 최적화 없음
- 현재 위치 기준 순차 탐색
- 단순한 탐색 전략

---

### 장점

- 구현이 가장 단순함
- 백트래킹 구조 이해 전 단계에서 탐색 과정 확인 가능

---

### 단점

- 불필요한 탐색이 많음
- 잘못된 선택을 빠르게 발견하지 못함
- 큰 문제에서는 탐색 공간 폭발 가능

---

# 2. Basic Backtracking

## 개념

Naive Search에 백트래킹을 적용한 방식입니다.

현재 위치에서 가능한 숫자만 선택하고,
잘못된 선택이면 이전 상태로 돌아가 다른 선택을 시도합니다.

---

동작 과정:

```

빈 칸 선택
↓
가능한 후보 계산
↓
후보 선택
↓
재귀 탐색
↓
실패하면 되돌리기

```

---

## 핵심 개념

백트래킹은:

> "현재 선택이 이후 해결 가능성을 막는다면 취소한다"

라는 전략입니다.

예:

```

3 선택

↓

다음 칸에서 불가능

↓

3 제거

↓

다른 후보 선택

```

---

## 특징

- 후보 계산 사용
- 잘못된 경로 제거
- 탐색 공간 감소

---

## 장점

- Naive보다 훨씬 적은 탐색
- 대부분의 Sudoku 해결 문제에서 사용되는 기본 구조

---

# 3. MRV (Minimum Remaining Values)

## 개념

Basic Backtracking에
변수 선택 최적화를 추가한 방식입니다.

MRV는:

> "가능한 후보가 가장 적은 칸부터 해결한다"

라는 전략입니다.

---

동작 과정:

```

모든 빈 칸 확인

↓

각 칸의 후보 계산

↓

후보가 가장 적은 칸 선택

↓

Backtracking 진행

```

---

예:

현재 상태:

```

A 칸
후보: 1,2,3
(3개)

B 칸
후보: 4
(1개)

C 칸
후보: 2,3
(2개)

```

MRV 선택:

```

B 칸

```

이유:

가능성이 적은 칸을 먼저 해결하면
잘못된 경로를 더 빨리 발견할 수 있기 때문입니다.

---

## 특징

- 후보 계산 사용
- 가장 제한된 변수 선택
- 탐색 트리 크기 감소

---

## 장점

- 불필요한 탐색 감소
- 백트래킹 성능 향상
- Sudoku와 같은 제약 만족 문제(CSP)에 효과적

---

# Algorithm Comparison

| 알고리즘 | 후보 계산 | 선택 전략 | Backtracking |
|---|---|---|---|
| Naive | X | 첫 번째 빈 칸 | X |
| Basic Backtracking | O | 첫 번째 빈 칸 | O |
| MRV | O | 후보 최소 칸 | O |

# Why MRV is Faster?

MRV는 탐색 순서를 개선합니다.

일반 Backtracking:

```

아무 칸 선택
↓
틀림
↓
되돌림

```

MRV:

```

가장 어려운 칸 선택
↓
빠르게 모순 발견
↓
불필요한 탐색 제거

```

결과적으로:

- 재귀 호출 감소
- 백트래킹 감소
- 해결 시간 감소

효과가 발생합니다.
